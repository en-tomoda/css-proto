# CSAS（キャリアセレクタビリティシステム）仕様書

> あなたの強みに気づく。あなたとキャリアを築く。

AIとの対話を通じてメンバーのキャリア目標設定・アクション管理を支援し、上司がメンバーの状態を（本人の承認のもとで）把握できる社内向けWebアプリケーション。本ドキュメントはモック実装（本リポジトリ）の現状仕様を反映したもの。

- 正式名称: キャリアセレクタビリティシステム
- 略称: CSAS
- ロゴ: enロゴ（`public/en-logo-trimmed.png`）、TAセクションはen TALENT ANALYTICSロゴ（`public/ta-logo-trimmed.png`）

## 1. 技術構成

| 項目 | 内容 |
|---|---|
| フレームワーク | Next.js 16（App Router / Turbopack） |
| UI | Tailwind CSS v4 + shadcn/ui（radix-novaスタイル） |

### デザイン方針
- コーポレートブルー基調のビジネス寄りトーン（角ゴシック体、落ち着いた配色、控えめな角丸）
- 文言は丁寧語ベース。ただし上司→メンバーの開示リクエストは意図的に「みたい」というやわらかいラベルを使用
- ユーザー画面（ログイン〜マイキャリア〜チャット〜履歴）は**最低425px幅**までのレスポンシブ対応

## 2. ユーザー権限とログイン

### ログイン画面（/login）
デモ用にログインユーザー（＝ロール）を選択する方式。ログアウトはヘッダー右上のアイコンから。

| デモの選択肢 | ロール | 挙動 |
|---|---|---|
| 田中 悠人（初回体験） | メンバー | 初回設定をリセットしてオンボーディングから体験 |
| 田中 悠人（メンバー） | メンバー | マイキャリアへ（初回設定完了済み想定） |
| 佐藤 拓也（上司） | 上司 | マイキャリアへ |
| 管理 太郎（管理者） | 管理者 | 管理者設定へ |

### 権限（複数保持可）
アカウントは複数の権限を持てる。**メンバーと上司は兼任不可**。許可される組み合わせ:

- メンバー / 上司 / 管理者（単独）
- メンバー ＋ 管理者
- 上司 ＋ 管理者

## 3. 初回設定（/onboarding）

メンバーの初回ログイン時のフロー。**チュートリアル → ブリッジ → 一問一答（5問） → 目標・アクション提案**の順。ボタン位置が動かないよう各ステップの高さは固定。カード枠のない背景直乗せUIで、チュートリアルは左右スライドで切り替わる。

### 3.1 チュートリアル（3スライド）
1. **CSASへようこそ** — アプリの位置づけとキャッチコピー
2. **CSASでできること** — 目標と今週のアクション / AIチャット / 自己理解レポート（TA）の3機能紹介
3. **情報はあなたがコントロール** — 入力内容は本人だけのもので、上司の「みたい」に対して開示可否を自分で決められること、いつでもロックに戻せることを説明

各スライドに「次へ / 戻る / スキップ」。

### 3.2 ブリッジ
「まずは、あなたのことを教えてください」— 質問の目的と所要時間の案内。

### 3.3 一問一答（5問）
1画面1質問。選択肢ボタン（タップ回答）と自由入力を併用。「前の質問に戻る」で修正可。

1. 自己紹介（どんな人か）
2. 大切にしている価値観
3. 挑戦したいこと
4. 避けたいこと
5. 進みたいキャリアパス（管理者設定のキャリアパスから選択）

### 3.4 提案
回答をもとに**仮目標**と**今週のアクション3件**を提示 → 「この内容で始める」でマイキャリアへ（完了トースト）。

## 4. マイキャリア（/mypage）

### 4.1 お知らせ（折りたたみ可）
ヘッダークリックで最小化/展開。件数バッジ付き。表示される通知:
- **上司からの「みたい」リクエスト** — 「佐藤 拓也さん（上司）があなたの『目標』を『みたい』と言っています」→「開示する / 見送る」
- **目標変更後のロック確認** — 目標を更新すると通知（開示中の情報があれば「ロックする / そのまま」、なければ「開示状況を確認しましょう」）
- **停滞アクションの深掘り** — 4週継続したアクションについてAIチャットへ誘導
- 定型のお知らせ（今週のアクション到着、先週の振り返り記録）

### 4.2 いまの目標
- 目標を強調ボックスで表示、目指すキャリアをバッジ表示
- **編集ボタン** — ダイアログで目標文とキャリアパスを更新。更新は目標の変遷として履歴に記録され、ロック確認通知が届く
- **変遷リンク** — /history へ

### 4.3 今週のアクション
- 週3件のTODO。チェックで完了、進捗バー表示
- 3件完了 → 翌週新しいアクション提示 / 未完了 → 継続（「◯週目・継続中」バッジ）/ 4週継続 → 深掘り通知
- **個別キャンセル** — 各アクションの✕から、理由（物理的に難しい／優先度が合わない／気が進まない／その他）を選択＋自由記述で取り下げ。キャンセルすると代わりのアクションが自動提案され、履歴に理由付きで記録される
- **アクション履歴リンク** — /history へ

### 4.4 振り返りの記録
週次振り返りの一覧。各行クリックで**AIチャットの要約モーダル**（話したこと / 気づいたこと / 次の一歩）を表示。「今週の振り返りをする」でチャットへ、「これまでの軌跡を見る」で /history へ。

### 4.5 上司に見せている情報
3種の開示状態（目標 / 達成したアクション / 現在チャレンジ中のアクション）を一覧化。開示済みの項目は**「ロックする」**でいつでも非公開に戻せる。

### 4.6 自己理解レポート（TA）
セクション7参照。

## 6. メンバー一覧・詳細（上司のみ）

### 開示承認制（「みたい」フロー）
- 対象: **目標 / 達成したアクション（過去の実績）/ 現在チャレンジ中のアクション**
- 状態: 未開示（locked）→ 承認待ち（requested）→ 開示済み（approved）
- 上司が「みたい」を送る → メンバーのマイキャリアに通知 → 「開示する」で開示（各操作でトースト）

### 関係値（ゲーミフィケーション）
- 開示情報数（0〜3）を**プログレスバー+%**（0/33/67/100%）
- レベル名: はじめまして → 顔見知り → 信頼関係 → パートナー

### メンバー一覧（/members）
カード形式。名前・TAタイプバッジ・関係値バー・3種の開示状態（「みたい」ボタン付き）。

### メンバー詳細（/members/[id]）
- **デフォルト表示（承認不要）**: 基本情報、TAタイプ、AIサマリー＋接し方のヒント、アプリ活用状況
- **アプリ利用状況ヒートマップ** — 直近の週×曜日のログイン濃淡（横=週の流れ・上に月ラベル、縦=曜日、正方形セルでカード幅いっぱい、月ラベルは1月のみ年付き）
- **活用状況**: ログイン日数 / AIチャット利用回数・頻度 / アクション達成数
- **AIサマリー** — 開示されている情報のみを利用して生成
- **アンロックで表示**: 目標 / 過去の実績 / チャレンジ中のアクション（未開示はロックカード＋「みたい を送る」）

## 8. これまでの軌跡（/history）

ナビには出さず、マイキャリアの各リンクから遷移。メンバー本人の記録を3セクションでまとめる。

1. **目標の変遷** — 設定・更新した目標のタイムライン（日付・現在バッジ・キャリアパス）
2. **アクションの履歴** — 達成したアクションとキャンセルしたアクション（理由付き）を新しい順に一覧
3. **振り返りの履歴** — 週次振り返りの記録。各行クリックでAIチャット要約モーダル

## 9. 管理者設定（/admin）

左サイドナビゲーション構成（狭幅では横スクロールに切替）。管理者はマイキャリア・AIチャットのナビ非表示。重要操作は確認モーダルを挟み、完了時にトーストを表示。

### 9.1 アカウント管理
- 一覧（氏名＋メール / 部署 / 権限 / 上司 / 状態 / 操作）。横スクロールしないよう列を最適化
- **検索**（氏名・メール・部署）＋**権限フィルター**（すべて/メンバー/上司/管理者）
- **ページネーション**（1ページ20件）
- **登録・更新** — ダイアログで氏名・メール・役割・部署を編集。役割はラジオ（メンバー/上司/なし）＋チェックボックス（管理者権限）で、メンバーと上司の兼任不可を担保。更新は確認モーダル
- **上司の紐付け** — メンバー権限を持つ行で、上司権限アカウントから担当上司を選択
- **利用停止 → 削除** — 削除ではなくまず利用停止（確認モーダル）。停止中のみ「利用再開」「完全削除（確認モーダル）」が可能
- 権限バッジは色分け（メンバー=グレー / 上司=ティール / 管理者=インディゴ）

### 9.2 キャリアパス設定
メンバーの目標設定（初回設定・AIチャット）の選択肢になるキャリアパスの追加・削除。

### 9.3 AIカスタマイズ
- ベースは「エン標準（CSAベース）」
- 個社ごとの昇進要件をテキストで設定（保存は確認モーダル）。AIが目標設定・相談時に参照する想定

デモの上司は**佐藤 拓也**（上司＋管理者）。他に渡辺・長谷川・井上などの上司、複数部署、招待中・利用停止の状態バリエーションを含む。検索・フィルター・ページネーションの動作確認用。

1. 上司（佐藤拓也）でログイン → メンバー一覧で田中さんに「みたい」を送る
2. ログアウト → メンバー（田中悠人）でログイン → マイキャリアのお知らせで「開示する」
3. 上司に戻る → 情報がアンロックされ関係値%が上がっている

## 12. モックの割り切り（本実装時の課題）

- AI応答・要約はすべて固定文（実装時はLLM連携）
- データ永続化なし（実装時はDB + 認証）
- 週次サイクル（アクション更新・深掘り通知）は状態で擬似表現（実装時はバッチ/スケジューラ）
- TA情報・利用状況ヒートマップは手入力／擬似生成のダミー（実装時はTalent Analytics・実ログ連携）
