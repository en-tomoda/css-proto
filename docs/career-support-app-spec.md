# エン キャリセレAI 仕様書

> 社員を主人公にする、キャリア自律支援ツール

AIとの対話を通じてメンバーのキャリア目標設定・アクション管理を支援し、上司がメンバーの状態を（本人が公開した範囲で）把握できる社内向けWebアプリケーション。本ドキュメントはモック実装（本リポジトリ）の現状仕様を反映したもの。

- 名称: **エン キャリセレAI**（旧称: キャリアセレクタビリティシステム / CSAS。評価指針としての「CSA」は本文で引き続き使用）
- ロゴ: enロゴ（`public/en-logo-trimmed.png`）、TAセクションはen TALENT ANALYTICSロゴ（`public/ta-logo-trimmed.png`）
- AIマスコット: **ナビ**（全身ピクトグラム調ロボット。マイキャリアの主役として登場）

## 1. 技術構成

| 項目 | 内容 |
|---|---|

| UI | Tailwind CSS v4 + shadcn/ui（radix-novaスタイル） |

### デザイン方針
- コーポレートブルー基調のビジネス寄りトーン（落ち着いた配色、控えめな角丸）に、AIマスコット「ナビ」で伴走感を添える
- 文言は丁寧語ベース
- ユーザー画面（ログイン〜マイキャリア〜AIトーク〜履歴）は**最低425px幅**までのレスポンシブ対応。管理者設定は横幅を広く使う（スマホ想定なし）

## 2. ユーザー権限とログイン

### ログイン画面（/login）
デモ用にログインユーザー（＝ロール）を選択する方式。ログアウトはヘッダー右上のアイコンから。

| デモの選択肢 | ロール | 挙動 |
|---|---|---|
| 田中 悠人（初回体験） | メンバー | 初回設定をリセットしてオンボーディングから体験 |
| 田中 悠人（メンバー） | メンバー | マイキャリアへ（初回設定完了済み想定） |
| 佐藤 拓也（上司） | 上司 | マイキャリアへ |
| 管理 太郎（管理者） | 管理者 | 管理者設定へ |

### 権限（複数保持可）
アカウントは複数の権限を持てる。**メンバーと上司は兼任不可**。許可される組み合わせ:

- メンバー / 上司 / 管理者（単独）
- メンバー ＋ 管理者
- 上司 ＋ 管理者

### 権限ごとに見える画面（ヘッダーナビ）

| 画面 | メンバー | 上司 | 管理者 |
|---|---|---|---|
| 初回設定 | ○（初回のみ） | ○ | - |
| マイキャリア | ○ | ○ | - |
| AIトーク | ○ | ○ | - |
| これまでの軌跡（/history） | ○ | ○ | - |
| メンバー一覧・詳細 | × | ○ | - |
| 管理者設定 | × | × | ○ |

管理者ロールのナビは「管理者設定」のみ。ヘッダー右にはお知らせベル（後述）・ロール名バッジ・ログアウトを表示。

## 3. お知らせ（ヘッダーのベル・全画面共通）

お知らせはヘッダー右上の**ベルアイコン**に集約（`components/header-notices.tsx`）。件数バッジ付き、クリックでポップオーバー表示。ロールに応じて内容が変わる。

- **メンバー**
  - **目標変更後の公開確認** — 目標を更新すると通知。公開中の情報があれば「非公開にする / そのまま」、なければ「確認しました」
  - **停滞アクションの深掘り** — 4週継続したアクションについてAIトークへ誘導（「AIに相談する」）
  - 定型のお知らせ（今週のアクション到着、先週の振り返り記録）
- **上司**
  - **メンバーが情報を非公開に戻した通知** — 「◯◯さんが『目標』を非公開に戻しました」＋理由。既読（×）で消去
- **管理者**: 通知対象がないためベル非表示

## 4. 初回設定（/onboarding）

メンバーの初回ログイン時のフロー。**チュートリアル → ブリッジ → 一問一答（5問） → 目標・アクション提案**の順。ボタン位置が動かないよう各ステップの高さは固定。カード枠のない背景直乗せUIで、チュートリアルは左右スライドで切り替わる。

### 4.1 チュートリアル（3スライド）
1. **エン キャリセレAIへようこそ** — アプリの位置づけとキャッチコピー
2. **エン キャリセレAIでできること** — 目標と今週のアクション / AIトーク / 自己理解レポート（TA）の3機能紹介
3. **情報はあなたがコントロール** — 入力内容・目標・アクションは本人だけのもので、公開可否を自分で決められること、いつでも非公開に戻せることを説明

各スライドに「次へ / 戻る / スキップ」。

### 4.2 ブリッジ
「まずは、あなたのことを教えてください」— 質問の目的と所要時間の案内。

### 4.3 一問一答（5問）
1画面1質問。選択肢ボタン（タップ回答）と自由入力を併用。「前の質問に戻る」で修正可。

1. 自己紹介（どんな人か）
2. 大切にしている価値観
3. 挑戦したいこと
4. 避けたいこと
5. 進みたいキャリアパス（管理者設定のキャリアパスから選択）

### 4.4 提案
回答をもとに**仮目標**と**今週のアクション3件**を提示 → 「この内容で始める」でマイキャリアへ（完了トースト）。

## 5. マイキャリア（/mypage）

### 5.1 ナビから見たあなた（デジタルツイン・主役カード）
`components/digital-twin-hero.tsx`。AIマスコット「ナビ」から見た本人像を表示する主役カード。

- **ナビのマスコット＋吹き出し** — 5種のセリフを数秒ごとに切り替えて「話している」ように見せる（吹き出しの高さは固定）
- **プロフィールの解像度** — %ではなく**4段階**（ぼんやり → うっすら → くっきり → 鮮明）＋ピクセル風ゲージで表現。会話・振り返り・アクションが増えるほど解像度が上がる想定
- **人物像** — 一言キャッチ（headline）＋プロフィール文（portrait）＋キーワードタグ
- **いま高めているCSA / 課題に感じているCSA** — CSA項目タグ（クリックで定義をポップオーバー表示、後述）
- **まだ、言語化されていないあなた** — ナビが投げかける問い＋CTA「ナビと話して解像度を上げる」→ AIトークへ

### 5.2 いまの目標
- 目標を強調ボックスで表示、目指すキャリアをバッジ表示
- **編集ボタン** — ダイアログで目標文とキャリアパスを更新。更新は目標の変遷として履歴に記録され、公開確認の通知が届く
- **変遷リンク** — /history へ

### 5.3 今週のアクション
- 週3件のTODO。チェックで完了、進捗バー表示。各アクションに**CSAタグ**（そのアクションが高めるCSA項目）を表示
- 3件完了 → 翌週新しいアクション提示 / 未完了 → 継続（「◯週目・継続中」バッジ）/ 4週継続 → 深掘り通知
- **個別キャンセル** — 各アクションの✕から、理由（物理的に難しい／優先度が合わない／気が進まない／その他）を選択＋自由記述で取り下げ。キャンセルすると代わりのアクションが自動提案され、履歴に理由付きで記録される
- **アクション履歴リンク** — /history へ

### 5.4 振り返りの記録
週次振り返りを**カレンダー表示**（`components/reflection-calendar.tsx`）。記録のある週をクリックで**AIトークの要約モーダル**（話したこと / 気づいたこと / 次の一歩）を表示。「今週の振り返りをする」でAIトークへ、「これまでの軌跡を見る」で /history へ。

### 5.5 上司への公開設定
4種の公開状態を一覧化し、**項目ごとに公開/非公開をメンバー自身が切り替える**。上司には公開した情報だけが見える。

- 目標 / 達成したアクション（過去の実績）/ 現在チャレンジ中のアクション / **振り返りの要約**
- **初期値はすべて非公開**。「公開する」で公開（確認モーダル）、「非公開にする」で理由を選んで非公開に戻す（理由は上司へ通知）

### 5.6 自己理解レポート（TA）
セクション8参照。

## 7. メンバー一覧・詳細（上司のみ）

### 情報公開の仕組み
- 対象: **目標 / 達成したアクション（過去の実績）/ 現在チャレンジ中のアクション / 振り返りの要約**
- **メンバー自身が公開/非公開を管理**（マイキャリアの「上司への公開設定」）。初期値は非公開
- 上司側は公開された情報のみ閲覧可。未公開はロック表示（上司からのリクエスト＝旧「みたい」フローは廃止、関係値表示も廃止）

### メンバー一覧（/members）
カード形式。名前・TAタイプバッジ・4種の公開状態（公開中は内容の要約、非公開は「🔒非公開」）。各カードから詳細へ。

### メンバー詳細（/members/[id]）
- **常時表示**: 基本情報、TAタイプ、AIサマリー＋接し方のヒント、アプリ活用状況
- **アプリ利用状況ヒートマップ** — 直近の週×曜日のログイン濃淡（横=週の流れ・上に月ラベル、縦=曜日、正方形セルでカード幅いっぱい、月ラベルは1月のみ年付き）
- **活用状況**: ログイン日数 / AIトーク利用回数・頻度 / アクション達成数
- **AIサマリー** — 公開されている情報のみを利用して生成
- **公開されると表示**: 目標 / 過去の実績 / チャレンジ中のアクション / 振り返りの要約（未公開はロックカード。本人が公開すると表示される旨を案内）

## 9. CSA（キャリアセレクタビリティ指針）

AIの評価・助言のベースとなる行動指針。ユーザー画面（マイキャリアのアクション・デジタルツイン）と管理者のAIカスタマイズで共通利用。

- **エン標準（27項目）**: 先頭7項目が「**考え方**」（自己変革性・目標必達性・多様受容性・周辺変革性・主観正義性・自発利他性・Enjoy-Thinking）、8項目め以降が「**能力**」（好感演出力〜新規事業創出力）
- **自社設定**: 個社独自の評価項目を追加できる（初期サンプル: 顧客起点力 / スピード実行力）
- **CSAタグ（`components/csa-tag.tsx`）**: 項目名バッジをクリックすると定義をポップオーバー表示。強化中（focus）/課題（challenge）/通常で色分け

## 10. これまでの軌跡（/history）

ナビには出さず、マイキャリアの各リンクから遷移。メンバー本人の記録を3セクションでまとめる。

1. **目標の変遷** — 設定・更新した目標のタイムライン（日付・現在バッジ・キャリアパス）
2. **アクションの履歴** — 達成したアクションとキャンセルしたアクション（理由付き）を新しい順に一覧
3. **振り返りの履歴** — 週次振り返りの記録。各行クリックでAIトーク要約モーダル

## 11. 管理者設定（/admin）

左サイドナビゲーション構成。横幅を広く使う。重要操作は確認モーダルを挟み、完了時にトーストを表示。

### 11.1 アカウント管理
- **一覧の列**: 氏名＋メール / 部署 / 権限 / 上司 / **キャリア希望** / 利用状況（アクション達成数）/ **成長支援度** / 直近の評価 / 状態 /（編集モード時のみ）操作
- **編集モード** — 既定はラベル表示。「編集」ONで上司・評価・状態などを変更可能
- **クイック絞り込み** — フリーワード検索（氏名・メール・部署）＋権限フィルター
- **詳細検索モーダル** — 部署 / 状態 / 上司 / キャリア希望 / 直近の評価 / 成長支援度 を **AND** で絞り込み。一致件数をリアルタイム表示、一括クリア。適用中は件数バッジと状況バーを表示
- **組み合わせソート** — 利用状況 / 成長支援度 / 直近の評価 の列見出しをクリックで **降順 → 昇順 → 解除** をトグル。複数列は**クリック順を優先度**にAND結合（優先度番号バッジ）。状況バーに並び順を明示＋一括クリア
- **成長支援度** — 上司がメンバーの成長をどれだけ支援できているかの度合い。**5段階（S/A/B/C/D ＝ 100/75/50/25/0）**。旧「関係値（部下）」は廃止
- **アカウント詳細モーダル** — プロフィール（キャリア希望 / 利用状況 / 成長支援度 / 直近の評価）＋「アカウント更新」タブ。前/次のメンバーへ移動可
- **ページネーション**（1ページ20件）
- **登録・更新** — 氏名・メール・役割・部署を編集。役割はラジオ（メンバー/上司/なし）＋チェックボックス（管理者権限）で、メンバーと上司の兼任不可を担保。更新は確認モーダル
- **上司の紐付け** — メンバー権限を持つ行で、上司権限アカウントから担当上司を選択
- **利用停止 → 削除** — まず利用停止（確認モーダル）。停止中のみ「利用再開」「完全削除（確認モーダル）」が可能
- 権限バッジは色分け（メンバー=グレー / 上司=ティール / 管理者=インディゴ）

### 11.2 キャリアパス設定
メンバーの目標設定（初回設定・AIトーク）の選択肢になるキャリアパスの追加・削除。

### 11.3 AIカスタマイズ
AIが目標設定・キャリア相談で参照する評価・指針のベースを設定。**CSA標準（エン標準）**と**自社設定**の2ブロック。各ブロックは開閉アニメーション付きの**アコーディオン**。

- **CSA標準（エン標準）** — 27項目を「考え方（1〜7）」「能力（8〜27）」の見出しで区切って表示。各項目に**チェックボックス**で活性/非活性。ヘッダーの親チェックで**全ON/全OFF**、「すべて選択 / すべて取り外す」の一括操作も可
- **自社設定** — 同様のアコーディオン。項目ごとのチェックボックス＋削除、項目の追加フォーム、一括操作
- 「参照する項目が1つでもある」ブロックを利用中とみなす。両方利用時は自社設定を優先しCSA標準を補完に使う旨を表示
- 保存は確認モーダル

## 12. ダミーデータ

### メンバー（5名）
初期の情報公開状況はデモ簡便のため全員**非公開**（マイキャリアから各自で公開/非公開を切り替え）。

| 名前 | TAタイプ | 特徴 |
|---|---|---|
| 田中 悠人 | フロントランナー | ログインユーザー本人。4週停滞アクション・目標変遷・キャンセル履歴あり |
| 佐藤 美咲 | インベスティゲーター | 高活用のスペシャリスト志向 |
| 鈴木 大輝 | サポーター | 5週停滞・低利用頻度（フォロー対象） |
| 高橋 結衣 | ディレクター | 高活動量・新規事業志向 |
| 伊藤 健 | ディフェンダー | 安定志向 |

各メンバーに、デジタルツイン（人物像・解像度・強化中/課題CSA・問い）、振り返り履歴（AIトーク要約付き）、目標変遷、キャンセル履歴を保持。

### アカウント（28件）
デモの上司は**佐藤 拓也**（上司＋管理者）。他に渡辺・長谷川・井上などの上司、複数部署、招待中・利用停止の状態バリエーションを含む。検索・詳細検索・ソート・ページネーションの動作確認用。各アカウントに直近の評価（S〜D）・担当上司・（上司が紐づく場合）成長支援度のモック値を保持。

### デモの推奨フロー
1. メンバー（田中悠人）でログイン → マイキャリアの「上司への公開設定」で目標などを「公開する」
2. ログアウト → 上司（佐藤拓也）でログイン → メンバー一覧・詳細で公開された情報が見える
3. メンバーに戻って一部を「非公開にする」（理由選択）→ 上司のヘッダーお知らせに通知が届く

## 14. モックの割り切り（本実装時の課題）
