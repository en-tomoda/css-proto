# CSAS（キャリアセレクタビリティシステム）仕様書

> あなたの強みに気づく。あなたとキャリアを築く。

AIとの対話を通じてメンバーのキャリア目標設定・アクション管理を支援し、上司がメンバーの状態を（本人の承認のもとで）把握できる社内向けWebアプリケーション。本ドキュメントはモック実装（本リポジトリ）の現状仕様を反映したもの。

- 正式名称: キャリアセレクタビリティシステム
- 略称: CSAS
- ロゴ: enロゴ（`public/en-logo-trimmed.png`）、TAセクションはen TALENT ANALYTICSロゴ（`public/ta-logo-trimmed.png`）

---

## 1. 技術構成

| 項目 | 内容 |
|---|---|
| フレームワーク | Next.js 16（App Router / Turbopack） |
| UI | Tailwind CSS v4 + shadcn/ui（radix-novaスタイル） |

### デザイン方針
- 「伴走するAI」のやわらかいトーン。丸ゴシック系フォント、大きめの角丸、ティール系カラー
- 文言も親しみやすく（例:「未開示」→「まだ見せてもらっていません」）
- ユーザー画面は**最低425px幅**までのレスポンシブ対応（ヘッダーは狭幅でアイコンのみ表示）

## 3. 初回設定画面（/onboarding）

AIとの**チャット形式**で仮目標と最初のアクションを設定する（メンバーの初回ログイン時のみ）。

質問フロー（5問）:
1. あなたはどんな人？（自己紹介）
2. 仕事における価値観は？
3. やりたいことは？
4. やりたくないことは？
5. 次に進みたいキャリアパスは？（管理者が設定したキャリアパスから選択式）

完了後、AIが**仮目標**と**今週のアクション3件**を提案 → 「この内容で始めてみる」で確定しマイページへ。

---

## 4. マイページ（/mypage）

- **いまの目標** — 目標とすすみたい道（キャリアパス）
- **今週のアクション** — 週3件のTODO。チェックで完了、進捗バー表示
  - 3件すべて完了 → 翌週新しいアクションを提示
  - 未完了あり → 継続（「◯週目 こつこつ継続中」バッジ）
  - **4週間（1ヶ月）継続** → 「ちょっと立ち止まっているアクションがあるみたい」通知カードを表示し、AIチャットでの深掘りへ誘導
- **上司からの「みたい」通知** — 開示リクエストが届くとカード表示。「見せてOK」/「今はやめておく」で応答
- **じぶんを知るヒント（TA）** — 下記セクション7参照

## 6. メンバー一覧・詳細（上司のみ）

### 開示承認制（「みたい」フロー）
- 対象情報: **目標 / 達成したアクション（過去の実績）/ 現在チャレンジ中のアクション** の3種
- 状態: 未開示（locked）→ お返事待ち（requested）→ 開示済み（approved）
- 上司が「👁 みたい」ボタンを送る → メンバーのマイページに通知 → 「見せてOK」で開示

### 関係値（ゲーミフィケーション）
- 開示された情報数（0〜3）を**プログレスバー+%表示**（0/33/67/100%）
- レベル名: はじめまして → 顔見知り → 信頼関係 → パートナー

### メンバー一覧（/members）
- カード形式。名前・TAタイプバッジ・関係値バー・3種の開示状態（「みたい」ボタン付き）

### メンバー詳細（/members/[id]）
- **デフォルトで見える情報（承認不要）**: 基本情報、TA情報、アプリ活用状況（ログイン日数・AIチャット利用回数/頻度・アクション達成数）
- **AIからのひとことサマリー** — アンロック済みの情報のみを利用して生成。接し方のヒント付き
- **アンロックで見える情報**: 目標 / 過去の実績 / チャレンジ中のアクション（未開示はロックカード+「みたい を送る」）

## 7. TA（Talent Analytics）セクション

実物のTalent Analytics受検結果レポートに準拠した構成。カードタイトル横にTALENT ANALYTICSロゴを表示。

| 項目 | 内容 | 表示 |
|---|---|---|
| タイプ | 認知特性から導かれる8タイプ | グラデーションカードで目立たせて表示 |
| 性格特性 | 主体性・変革性・外向性・持続性・協調性（偏差値） | レーダーチャート |
| キャリアタイプ指向性 | 経営幹部/アントレプレナー/チャレンジャー/自立/スペシャリスト/安定志向/私生活重視/社会奉仕（得点） | TOP3を🥇🥈🥉ランキング表示 |
| コミュニケーション力 | 意思伝達力・論理的表現力・好感表現力・対人調和力 | バー表示 |
| 創造的思考性 | 発想性（拡散思考）・論理性（収束思考） | バー表示 |
| 認知特性 | 知覚（事実⇄直観）・判断（論理⇄感覚） | 両側スライダー |
| 総合特徴 | やわらかい文体の要約 | まとめカード |

### 8タイプ一覧
| タイプ | 特徴 |
|---|---|
| インベスティゲーター | 情報を細かく確認し、論理的かつ主体的に考える追究派 |
| ディフェンダー | 情報を細かく確認し、論理的にフォローアップする守備派 |
| ディレクター | 要点を掴んで合理的かつ主体的に対応する指示役 |
| アドバイザー | 要点を掴んで合理的に相手に合わせて説明する相談役 |
| フロントランナー | 全体を把握して素早く積極的に動く行動派 |
| チームワーカー | 全体を把握して素早く周囲に合わせて動く追従派 |
| コーディネーター | 状況を細かく確認し、配慮の上で主体的に動くまとめ役 |
| サポーター | 状況を細かく確認し、配慮の上でフォローアップする支援者 |

## 8. 管理者設定（/admin）

左サイドナビゲーション構成（狭幅では横スクロールに切替）。管理者はマイページ・AIチャットのナビ非表示。

1. **アカウント管理** — ユーザーアカウント一覧、登録（ダイアログ）・削除。権限（メンバー/上司/管理者）と状態（利用中/招待中）
2. **キャリアパス設定** — メンバーの目標設定（初回設定・AIチャット）の選択肢になるキャリアパスの追加・削除
3. **AIカスタマイズ** — ベースは「エン標準（CSAベース）」。個社ごとの昇進要件をテキストで設定可能（AIが目標設定・相談時に参照する想定）

## 9. ダミーデータ

メンバー5名（開示状態・停滞アクション・利用頻度がそれぞれ異なる）:

| 名前 | TAタイプ | 開示状況 | 特徴 |
|---|---|---|---|
| 田中 悠人 | フロントランナー | 全ロック（デモの起点） | ログインユーザー本人。4週停滞アクションあり |
| 佐藤 美咲 | インベスティゲーター | 全開示（100%） | パートナー関係のサンプル |
| 鈴木 大輝 | サポーター | 目標のみ開示 | 5週停滞・低利用頻度（フォロー対象のサンプル） |
| 高橋 結衣 | ディレクター | 目標リクエスト中 | 高活動量 |
| 伊藤 健 | ディフェンダー | 全ロック | 安定志向 |

### デモの推奨フロー
1. 上司（佐藤拓也）でログイン → メンバー一覧で田中さんに「みたい」を送る
2. ログアウト → メンバー（田中悠人）でログイン → マイページで「見せてOK」
3. 上司に戻る → 情報がアンロックされ関係値%が上がっている

## 11. モックの割り切り（本実装時の課題）

- AI応答はすべて固定文（実装時はLLM連携）
- データ永続化なし（実装時はDB + 認証）
- 週次サイクル（アクション更新・深掘り通知）は状態で擬似表現（実装時はバッチ/スケジューラ）
- TA情報は手入力のダミー（実装時はTalent Analytics連携）
